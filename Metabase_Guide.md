# Руководство по работе с Metabase для мониторинга ЕГИСЗ

## Введение

Metabase - это инструмент для визуализации данных и создания дашбордов на основе SQL-запросов к базе данных PostgreSQL. В этом проекте Metabase используется для мониторинга обменов данными с ЕГИСЗ (Единой государственной информационной системой в сфере здравоохранения).

## Установка и настройка Metabase

1. **Запуск Metabase**: Metabase запускается как часть Docker Compose в контейнере `metabase`.
2. **Доступ**: После запуска перейдите по адресу `http://localhost:3000` (или настроенному порту).
3. **Первоначальная настройка**:
   - Выберите язык (рекомендуется русский).
   - Подключите базу данных PostgreSQL с параметрами из `docker-compose.yml`.
   - Создайте администратора.

## Структура данных

Данные хранятся в следующих таблицах:
- `fact_transactions`: Факты транзакций
- `dim_clinics`: Измерение клиник
- `dim_services`: Измерение сервисов
- `egisz_errors`: Ошибки ЕГИСЗ
- `v_unified_analytics`: Объединённое представление для аналитики

## Создание отчётов

### Шаг 1: Выбор источника данных
- В Metabase перейдите в раздел "Browse data" или "Обзор данных".
- Выберите базу данных `egisz_monitor` (или настроенное имя).

### Шаг 2: Создание SQL-запроса
- Выберите "Native query" (Собственный запрос) для написания SQL.
- Используйте готовые запросы из файлов:
  - `postgres/metabase/001_operational_metrics.sql` - Операционные метрики
  - `postgres/metabase/002_semantic_layer_metrics.sql` - Семантические метрики

### Шаг 3: Визуализация
- После выполнения запроса нажмите "Visualize" (Визуализировать).
- Выберите тип графика:
  - **Table** (Таблица) - для табличных данных
  - **Bar chart** (Столбчатая диаграмма) - для сравнения категорий
  - **Line chart** (Линейный график) - для трендов по времени
  - **Pie chart** (Круговая диаграмма) - для распределения
  - **Heatmap** (Тепловая карта) - для статусов

### Шаг 4: Сохранение и публикация
- Сохраните отчёт в коллекцию.
- Добавьте на дашборд для автоматического обновления.

## Основные отчёты

### 1. KPI за последние 24 часа
**Цель**: Общий обзор успешности обменов.
**SQL**: Используйте запрос 1 из `001_operational_metrics.sql`
**Визуализация**: Карточки (Cards) для показателей.

### 2. Распределение ошибок
**Цель**: Анализ типов ошибок.
**SQL**: Запрос 2 из `001_operational_metrics.sql`
**Визуализация**: Круговая диаграмма.

### 3. Почасовой тренд
**Цель**: Мониторинг активности по часам.
**SQL**: Запрос 3 из `001_operational_metrics.sql`
**Визуализация**: Линейный график с двумя линиями (успехи и ошибки).

### 4. Надёжность клиник
**Цель**: Идентификация проблемных клиник.
**SQL**: Запрос 4 из `001_operational_metrics.sql`
**Визуализация**: Таблица с сортировкой по количеству ошибок.

### 5. Здоровье сервисов
**Цель**: Мониторинг производительности сервисов.
**SQL**: Запрос 5 из `001_operational_metrics.sql`
**Визуализация**: Столбчатая диаграмма.

### 6. Тепловая карта активности
**Цель**: Визуальный статус всех клиник и сервисов.
**SQL**: Запрос 6 из `001_operational_metrics.sql`
**Визуализация**: Тепловая карта (зелёный/жёлтый/красный).

### 7. Связки ЮЛ и типов СЭМД
**Цель**: Анализ активности по юридическим лицам и типам документов.
**SQL**: Запрос 7 из `002_semantic_layer_metrics.sql`
**Визуализация**: Таблица или столбчатая диаграмма.

### 8. Типы ошибок по клиентам
**Цель**: Детальный анализ ошибок по каждой клинике.
**SQL**: Запрос 8 из `002_semantic_layer_metrics.sql`
**Визуализация**: Таблица с группировкой.

### 9. Топ ошибок по частоте
**Цель**: Идентификация наиболее распространённых проблем.
**SQL**: Запрос 9 из `002_semantic_layer_metrics.sql`
**Визуализация**: Таблица или столбчатая диаграмма.

### 10. Производительность по типам СЭМД
**Цель**: Мониторинг эффективности различных типов документов.
**SQL**: Запрос 10 из `002_semantic_layer_metrics.sql`
**Визуализация**: Столбчатая диаграмма с процентами.

## Коллекция дашбордов

### Создание коллекции "EGISZ Мониторинг"

1. В Metabase перейдите в раздел "Collections" (Коллекции)
2. Нажмите "+ New collection"
3. Назовите "EGISZ Мониторинг"
4. Создайте подколлекции:
   - "Операционные метрики"
   - "Семантические метрики"
   - "Аналитика ошибок"

### Дашборд "Главный мониторинг" (Main Dashboard)

**Цель**: Объединяет ключевые метрики для ежедневного мониторинга

**Компоненты**:
1. KPI карточки (запрос 1)
2. График тренда (запрос 3)
3. Топ проблемных клиник (запрос 4, топ 5)
4. Распределение ошибок (запрос 2)

**Фильтры**:
- Дата/время (последние 24 часа по умолчанию)
- Возможность выбора периода

### Дашборд "Анализ ошибок" (Error Analysis)

**Цель**: Детальный разбор всех типов ошибок

**Компоненты**:
1. Топ ошибок по частоте (запрос 9)
2. Ошибки по клиентам (запрос 8)
3. Отпечатки ошибок (запрос 1 из semantic)
4. Рост инцидентов (запрос 6 из semantic)

### Дашборд "Клинический анализ" (Clinic Analysis)

**Цель**: Мониторинг работы клиник

**Компоненты**:
1. Надежность клиник (запрос 4)
2. Тепловая карта активности (запрос 6)
3. SLA по клиникам (запрос 2 из semantic)
4. Непроверенные клиники (запрос 5 из semantic)

### Дашборд "Сервисный анализ" (Service Analysis)

**Цель**: Мониторинг сервисов и типов СЭМД

**Компоненты**:
1. Здоровье сервисов (запрос 5)
2. Связки ЮЛ и СЭМД (запрос 7)
3. Производительность по типам СЭМД (запрос 10)
4. Здоровье сервисов по часам (запрос 3 из semantic)

### Дашборд "Тренды и прогнозы" (Trends & Forecasting)

**Цель**: Анализ долгосрочных трендов

**Компоненты**:
1. Почасовой тренд (расширенный период)
2. Рост инцидентов по клиникам
3. Сравнение периодов (неделя к неделе)
4. Прогнозные метрики

## Создание дашборда

1. Перейдите в "Dashboards" (Дашборды).
2. Создайте новый дашборд.
3. Добавьте сохранённые вопросы (отчёты).
4. Настройте фильтры для интерактивности (например, по дате).
5. Опубликуйте дашборд для команды.

## Импорт готовых дашбордов

Для быстрого создания дашбордов используйте готовые JSON файлы в папке `metabase_dashboards/`.

### Инструкция по импорту:

1. В Metabase перейдите в раздел "Dashboards"
2. Нажмите "..." (три точки) в правом верхнем углу
3. Выберите "Import dashboard"
4. Выберите JSON файл из `metabase_dashboards/main_dashboard.json`
5. Выберите коллекцию для сохранения
6. Нажмите "Import"

### Доступные дашборды для импорта:

- `main_dashboard.json` - Главный мониторинг EGISZ (KPI + тренды + ошибки)
- `error_analysis_dashboard.json` - Анализ ошибок (топ ошибок + по клиентам + рост инцидентов)
- `clinic_analysis_dashboard.json` - Анализ клиник (надежность + активность + непроверенные)
- `service_analysis_dashboard.json` - Анализ сервисов (здоровье + ЮЛ/СЭМД + производительность)

### Настройка после импорта:

1. Проверьте подключение к базе данных
2. Обновите ID базы данных в настройках карточек (если отличается)
3. Настройте фильтры и параметры
4. Опубликуйте дашборд

## Troubleshooting

- **Ошибки подключения**: Проверьте параметры БД в `docker-compose.yml`.
- **Пустые результаты**: Убедитесь, что ETL процесс запущен и данные загружены.
- **Производительность**: Используйте индексы и ограничивайте временные диапазоны.

## Аудит данных

Для проверки корректности:
1. Сравните количества в отчётах с сырыми данными в таблицах.
2. Проверьте парсинг XML в логах ETL.
3. Валидируйте расчёты процентов и агрегаций.
